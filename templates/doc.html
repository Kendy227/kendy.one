Developer API Documentation

Empower your platform with our high-performance recharging API. Secure, reliable, and built for scale.

Authentication

All API requests require a pair of keys: public_key and private_key. These should be passed as query parameters or in the request body.

Security Warning

Never expose your private_key in client-side code. Always perform API calls from your server-side environment.

Get User Balance

GET POST /api/v2/user/balance

Retrieve real-time balance information for your account, including BRL and INR holdings.

Parameter Type Status Description
public_key String Required Your public API key
private_key String Required Your private API key
Response Example 
{
  "success": true,
  "balance": 7500.25,
  "is_reseller": 1,
  "username": "john_reseller",
  "email": "john@example.com"
}

Get Reseller Products

GET POST /api/v2/products

Fetch the complete catalog of active products with exclusive reseller pricing.

Response Example 
{
  "success": true,
  "product_count": 2,
  "products": [
    {
      "id": 1,
      "product_name": "MLBB 5000 Diamonds",
      "reseller_price": 450.50,
      "category": "MLBB"
    }
  ]
}

Create Order

POST /api/v2/create-order

Initiate a recharge transaction. Funds will be automatically deducted from your BRL wallet.

Parameter Type Status Description
userid String Required In-game User ID
zoneid String Required In-game Server/Zone ID
product_id Integer Required Database Product ID
Success Response 
{
  "success": true,
  "order_id": 12345,
  "status": "success",
  "price": 500.00
}

Validate Player

GET /api/v2/validate

Verify player existence and retrieve their in-game nickname before placing an order.

Response Example 
{
  "success": true,
  "username": "LegendaryPlayer",
  "country": "Brazil",
  "source": "smile.one"
}

Error Codes

Code Status Reason
401 Unauthorized Invalid API key pair provided
402 Payment Required Insufficient BRL balance
403 Forbidden IP not whitelisted or not a reseller
